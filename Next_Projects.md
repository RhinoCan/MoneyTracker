## Grocery List

Template (Pantry List): A list of regularly purchased items that may or may not be purchased on any given trip to the grocery store. The list could contain simply the name of item, e.g. margarine, with optional details such as quantity or size. The list begins empty and users add one item at a time. At any point, the list can be manually sorted to put it in the order that these things will be found at a given supermarket. That happens via an up-caret and a down-caret on each row. The list has two basic groups of items: food and supplies. Supplies are anything that you buy at the grocery store that isn't edible e.g. garbage bags. Each item has a checkbox or toggle in front of it and a "Create grocery list" button is provided at the bottom. When the "Create grocery list" button is pressed, a grocery list is generated. It should be possible to click on any item in the template and either edit it, delete it, or move it within the list.

Grocery List: The grocery list simply contains all of the items that were selected from the template, sorted in the same order. Again, each item contains a checkbox or toggle that the user can check to indicate that the item has been purchased. The user grabs items on the list as they go around the store and ticks each off from the Grocery List as they do so. When all items are ticked off, the user goes to the checkouts. The list contains a delete list button that the user can click upon completion of the list. (Maybe) It should be possible to click on any item in the list and either edit it, delete it or move it within the list without changing it in the template.

Can the user go back to the template after the list has been generated to add additional items? That should be possible so provide a button on the grocery list that returns to the template and allows more items to be added to the current list. When there is an existing grocery list, the "Create grocery list" button should say "Update grocery list". There is no provision to build multiple concurrent grocery lists - why would that be needed? - so it is always true that there is a current grocery list or that there isn't.


### Future Enhancements to Grocery List

Should multiple templates be possible? That could be useful if the user frequents multiple grocery stores and goes to each for different things.

Should the user be able to click on any item in the grocery list and either edit it, delete it, or move it within the grocery list without changing it in the template? I can imagine wanting to edit it - perhaps with respect to the quantity - or maybe move it but both seem to not be strictly necessary.

Provide the ability to assign individual items in the list to a specific person so that the list could be divided betwen family members. For example, Bob goes to the butcher and buys specific meats; Mary stops at the bakery for bread and desserts; Carol buys the other staples.

Provide Register and Login pages so that multiple users are possible with all data segregated by user (or family if multiple people participate in the shopping.)
