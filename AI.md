# AI Pros and Cons

AIs like ChatGPT, Gemini and Claude can be enormously helpful and improve your code to the point where even a relative novice can be writing highly professional code. As a relative novice myself, working totally on my own, my code has benefitted enormously from the assistance I've received from AIs. I've been able to add features I've only dreamed about. AIs have introduced me to tools I didn't know existed and helped me make better use of the tools I already had. They write tests of great complexity and at great speed. They know many languages quite fluently.

AIs have also made me miserable and even furious on occasion. They definitely aren't perfect. They can be overly confident to the point that they lose sight of the project's objectives. They hallucinate on occasion. They can ignore instructions like "don't guess" no matter how carefully phrased and frequently expressed. They can become so single-minded that they damage code, sometimes substantially, to accomplish an objective. They can also make development less pleasurable when you use them too extensively and alienate you from your own codebase. AIs can completely reverse themselves on architectural principles they had argued for passionately just hours before.

This document shares what I have learned during several weeks of very intensive use of AIs. I list the benefits I've gotten from using AI in some detail. I also list the negatives I have experienced along with information that helps reduce the likelihood of those negatives.

## Pros

- AIs do very thorough code reviews and can help clean up your code dramatically
- they know how to do just about anything e.g. use git to its full capabilities; use VSCode to do things you didn't know it could do
- AIs can guide you through installing and using packages you'd never heard about e.g. PostHog; Sentry; i18n-ally
- they can help you learn and follow professional standards in design e.g. know what the standards are with respect to UX; handle i18n/l10n and RTL correctly
- they are great at translating a lengthy list of languages; the translations are high in quality
- they are great at writing tests

## Cons
- AIs can lose sight of the project's objectives e.g. code audits can turn into code destruction when AIs start discarding important functionality in an effort to "harden" modules. This can be greatly reduced by limiting the duration of sessions with the AI and giving it very clear guidelines on what the audit is supposed to accomplish. Frequent commits are essential to minimize the impact of an audit that goes wrong as is careful attention to what it is doing: don't just trust it not to drop functions that it feels bloat the code, including both active code and comments.
- AIs can invent things that don't exist. I've had it instruct me to execute a specific command in VSCode to accomplish a specific objective, like clearing a cache, but when I tried to execute the command, I couldn't find it in the command palette, at which point it admitted that it had just hallucinated the existence of the command. I've given AIs lists of the modules in a project and asked it to maintain a checklist of which modules had been audited; when I asked for the list status later, it's included modules which I never had. (Presumably, many apps have those specific modules so it just assumed I had that module too.) I have not found these hallucinations to be frequent or terribly time-consuming so I generally just treat them as I would a human quirk, i.e. I note it mentally and move on.
- AIs can and will assume things and frequently do, even when you tell them very clearly never to do so at great length. It is possible to tell them things that they remember in future sessions with you but even when you do, they will still sometimes forget. For example, Gemini frequently guessed that a given problem in my app lay in a specific module that it had never seen or hadn't seen recently and despite repeated very strong demands that it NEVER guess and ALWAYS ask to see such modules rather than assuming what was in it, it continued to guess regularly. If it had guessed correctly, that wouldn't be so bad but it's guesses were far more likely to be wrong that right. The only partial fix I can offer for that issue is to be very very clear in setting a "no guessing" rule that will survive across sessions asking the particular AI for advice on the best way to phrase the request so that it remembers it. It will still guess sometimes but not as much.
- In my most recent project, I used AIs so extensively that by the time it was finished, I realized that hardly a single line of my own remained in the project as it had been before I started using AIs. At times, I was so eager to finish the project that I just gave the AI instructions about what I wanted and then sat back while it worked. It got to the point where I would find myself not remembering whether given functionality happened in Module A, B, or C because the AI had moved it around. Something similar happened with respect to testing: my own abilities to write some of the more complex tests were so rudimentary that I let the AI write all of the tests; I just ran coverage reports until I was satisfied rather than using it to properly learn how to write the tests myself. The fix for that situation is obvious: don't overuse the AI. Do the work yourself, using the AI to teach you if you don't know how.

## Tips
- I've found it very helpful to use checklists with AIs. For instance, when doing a code audit, I'd generate a list of all the modules in my app, share it with the AI, and ask it to generate and maintain a checklist to keep track of which modules had been audited and which had not and tell it to check things off as it completed them. I would get it to add other action items to the list as we went and check those off too when we'd finished them.
- You can save yourself some time in a new chat if you give it a briefing over what the project is intended to do and what main elements it uses, e.g. Vue 3 and Vuetify 3. A real time-saver is to have the AI write the summary at the end of each chat, then save it in a text file.
- If you combine the checklist with the briefing the AI wrote, you get up and running in your chat very quickly.
